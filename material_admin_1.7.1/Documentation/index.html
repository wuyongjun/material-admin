About Material Admin


Material Admin is a Google Material Design inspired powerful-lightweight fully responsive retina display compatible admin template based on latest Bootstrap 3. Material Admin comes with both jQuery and AngularJs versions.

Unless like other admin templates, all the plugin used in Material Admin are completely customized to match the Material Design.

About CSS files


All LESS files including third party's are compiled in to a single CSS file and, splitted in to 2 minified pieces in order to gain support for IE 9. In IE 9, a sheet may contain up to only 4095 selectors and anything above will be ignored.

Make sure to include both app.min.1.css and app.min.2.css in your project files in proper sequence.

NodeJs and NPM


Material Admin comes with GruntJs for automated tasks and a complete set of third party dependencies. Source files of these dependencies are not included in your downloaded directory and in order to install them, you'll need to have NodeJs and NPM installed in your computer. Make sure to install this duo before your proceed anything with your new project.

Installation


Make sure to install both NodeJs and NPM in your computer as mentioned in previous section, before you proceed anything in this section.


  1. Extract the file downloaded using any file archive utility software.
  2. Copy all the files and folders from template directory to your project directory.
  3. Install all the NPM dependencies by firing the following command in your main project directory where you can find package.json 
    npm install
  4. That's it. Now you can start previewing Material Admin.

GruntJs


GruntJs is a task-based command line build tool for JavaScript projects. It helps making most of our manual tasks to automate such as minifying, compiling, splitting etc. In Material Admin, GruntJs is been used for:

  1. Compiling LESS to CSS.
  2. Splitting main CSS file to sub instances.
  3. Minification of CSS and JS files
  4. Concating JS files.
  5. Cleaning Temp files.
  6. Template Caching (AngularJs).

All the above tasks are added in gruntfile.js which is available in the main project folder. gruntfile just creates a basic grunt.js that you can fill in with tasks.



Installing GruntJs


GruntJs is available as a NPM package and already included in our main NPM dependency list in package.json

Doing a simple:

npm install

as mentioned in 'NodeJs and NPM', will install all the dependencies including GruntJs.



Before you make any changes...


before you make any changes in less or template (Angularjs) directory files, make sure to run: 

grunt watch

grunt watch is a file watcher which will execute certain tasks when any given file changes. All less, js files and template files in angularjs variant should be watched in order to make changes in the output files (css and template.js).



In order to create a distributed package (jQuery version only)...


In order to create a distributed package which excludes all the dev related files, run the following Grunt command.

grunt dist

This will clean the project by removing all the temp files such as .DS_Store .idea and create a 'dist' folder with only the needed files to run and preview the template.
